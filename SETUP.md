# 🚀 Первоначальная установка проекта

Этот документ описывает, как получившему проект новому пользователю быстро всё поднять и запустить.

## Шаг 1: Клонирование и установка зависимостей

```bash
# Клонируйте репозиторий
git clone <ваш-репо>
cd <имя-проекта>

# Установите зависимости
pnpm install
```

## Шаг 2: Создание конфигурационного файла

Создайте файл `.env.development.local` в корне проекта:

```bash
touch .env.development.local
```

Откройте файл и добавьте необходимые переменные окружения. 

**Пример содержимого** (замените на ваши реальные значения):

```env
# Обязательно! Строка подключения к вашей PostgreSQL БД
# Получите это значение у администратора БД или из вашего хостинга (например, Neon, AWS, etc.)
DATABASE_URL="postgresql://neondb_owner:YOUR_PASSWORD@db.neon.tech:5432/neondb?sslmode=require"

# Обязательно! API ключ OpenAI для AI функций
# Получите на https://platform.openai.com/api-keys
OPENAI_API_KEY="sk-..."

# Обязательно! Генерируется один раз для безопасности сессий
# Сгенерируйте командой: openssl rand -base64 32
BETTER_AUTH_SECRET="your_secret_key_here"

# Опционально - переменные v0
# Эти переменные обычно устанавливаются v0 автоматически
AI_GATEWAY_API_KEY="vck_..."
V0_RUNTIME_URL="https://..."
V0_CALLBACK_URL="https://..."
```

## Шаг 3: Инициализация базы данных

Запустите единую команду, которая проверит всё и инициализирует БД:

```bash
pnpm setup
```

Эта команда:
- ✅ Проверит наличие `.env.development.local`
- ✅ Проверит, что установлены все обязательные переменные
- ✅ Автоматически применит миграции БД
- ✅ Покажет детальные сообщения об ошибках, если что-то не так

**Ожидаемый результат:**

```
🚀 Инициализация проекта

📋 Проверка конфигурации окружения...
✅ Найден .env.development.local

🔐 Проверка обязательных переменных окружения...

  ✅ DATABASE_URL - Подключение к БД
  ✅ OPENAI_API_KEY - API ключ OpenAI
  ✅ BETTER_AUTH_SECRET - Секретный ключ для авторизации

✅ Все обязательные переменные установлены!

🗄️  Инициализация базы данных...

⏳ Это может занять некоторое время...

✅ Проект успешно инициализирован!

Теперь вы можете запустить проект:

  pnpm dev
```

## Шаг 4: Запуск проекта

Если всё прошло успешно:

```bash
pnpm dev
```

Проект запустится на `http://localhost:3000`

## ⚠️ Решение проблем

### "DATABASE_URL не установлена"

**Причина:** Переменная окружения не найдена в `.env.development.local`.

**Решение:**
1. Убедитесь, что файл `.env.development.local` создан
2. Проверьте, что там есть строка: `DATABASE_URL="..."`
3. Убедитесь, что значение в кавычках (см. пример выше)
4. Запустите `pnpm setup` снова

### "Не удается подключиться к БД"

**Причина:** Значение `DATABASE_URL` неверно или БД недоступна.

**Решение:**
1. Проверьте, правильно ли скопирована строка подключения
2. Убедитесь, что БД запущена и доступна
3. Попробуйте подключиться с помощью `psql`:
   ```bash
   psql "ваша-DATABASE_URL"
   ```
4. Если не работает — проверьте с администратором БД

### "OPENAI_API_KEY не установлена"

**Причина:** API ключ OpenAI отсутствует.

**Решение:**
1. Перейдите на https://platform.openai.com/api-keys
2. Создайте новый ключ
3. Добавьте в `.env.development.local`:
   ```
   OPENAI_API_KEY="sk-..."
   ```
4. Запустите `pnpm setup` снова

### "BETTER_AUTH_SECRET не установлена"

**Причина:** Секретный ключ для сессий не создан.

**Решение:**
1. Сгенерируйте новый ключ:
   ```bash
   openssl rand -base64 32
   ```
2. Добавьте результат в `.env.development.local`:
   ```
   BETTER_AUTH_SECRET="<результат_команды_выше>"
   ```
3. Запустите `pnpm setup` снова

### "Порт 3000 уже занят"

**Причина:** Другой процесс использует порт 3000.

**Решение:**
```bash
# Используйте другой порт
pnpm dev -- -p 3001
```

## 📚 Дополнительно

### Как добавить новые миграции БД

Если нужно добавить новые таблицы или изменить существующие:

```bash
# 1. Отредактируйте lib/db/schema.ts

# 2. Сгенерируйте миграцию
pnpm db:generate

# 3. Примените миграцию
pnpm db:push
```

Подробнее см. [MIGRATIONS.md](./MIGRATIONS.md)

### Команды разработки

```bash
pnpm dev        # Запустить dev сервер
pnpm build      # Собрать проект
pnpm start      # Запустить production версию
pnpm lint       # Проверить код
pnpm setup      # Инициализировать БД и проверить конфиг
pnpm db:init    # Инициализировать БД вручную
pnpm db:migrate # Применить миграции
```
